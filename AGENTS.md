# AGENTS.md

## Purpose

このリポジトリは、`GitHub Pages` で公開するための demo アプリを作る。
テーマは `実習先口コミデータベース構築システム`。

目的は、単なる口コミ投稿サービスではなく、
`自由記述レビュー + インタビュー型補完` を体験できるプロトタイプを、短時間で理解できる形で見せること。

## Product Concept

demo アプリでは次の流れが一目で伝わることを優先する。

1. 実習先レビューを探す
2. 閲覧前に短いインタビューに回答する
3. 回答によって不足データが補完される
4. 構造化されたレビューと要約を閲覧する

## Build Policy

- `GitHub Pages` でそのまま配信できる静的フロントエンドを前提にする
- `GitHub Pages` の公開物は `/docs` に出力する
- まずは `フロントエンド完結のデモ` として作る
- バックエンドや認証はモックで表現する
- データはローカルの `JSON / TS` 定義で持つ
- LLM 処理は実 API 連携ではなく、`解析済み結果` と `質問生成結果` をモック表示してよい

## Current Demo Scope

- `React + Vite` で構築する
- 実習先サンプルは `A病院` `B病院` `C病院` の 3 件とする
- `B病院` は一部入力済み、一部未入力の状態で始まる
- `C病院` は全項目入力済みだが、`実習の雰囲気` と `指導医・スタッフ` は 2 年以上前の古い情報として扱う
- 初期表示では右カラムは未選択状態とし、操作案内のみを表示する
- 実習先一覧は `総合評価の高い順` に並べ、`N/A` は最後に配置する
- OpenAI API Key は起動時モーダルから入力し、`localStorage` のみに保存する
- API Key については `この画面から自動でネットワーク送信されない` 注意書きを明記する
- アイコンは `icon.jpg` を利用し、`favicon` と `PWA icon` の両方に使う
- 病院カードの左側には、外部アイコンライブラリの一般的な病院アイコンを表示する

## Screen Priorities

最初に作る画面はこの 3 つに絞る。

1. `レビュー一覧 / 実習先一覧`
2. `ログイン時インタビュー表示モーダル`
3. `実習先詳細ページ`

余力があれば以下を追加する。

- AI 要約カード
- 情報信頼度表示
- 古い情報の更新促進 UI

## UI Direction

デザインは `レビューアプリっぽさ` を明確に出す。

- 情報が一覧で比較しやすい
- 評価、タグ、要約、件数がすぐ見える
- `口コミサイト` と `ナレッジベース` の中間の見た目にする
- 学生向けだが、軽すぎず、調査ツールのような信頼感を持たせる

### Visual Keywords

- clean
- editorial
- review dashboard
- structured cards
- muted blue / slate / warm accent

### Avoid

- 企業管理画面に寄りすぎるデザイン
- 汎用 SaaS テンプレートそのままの見た目
- 派手すぎるアニメーション
- 情報量が少なく見える大きすぎる余白

## Recommended Information Architecture

### List Page

- 検索バー
- エリア or 診療科タグ
- 実習先カード一覧
- カード内に以下を表示
  - 病院アイコン
  - 実習先名
  - 所在地 or 最寄り
  - 総合評価
  - 雰囲気タグ
  - 課題量の目安
  - レビュー件数
  - AI 要約

### Interview Modal

- 「レビューを見る前に 30 秒だけ協力してください」の導線
- 1 問 1 答のテンポ良い UI
- 選択式を中心にする
- 回答進捗を表示する
- 回答後にレビュー閲覧を開放する

### Detail Page

- 実習先の概要
- AI 要約
- 構造化レビュー
  - 交通アクセス
  - スケジュール
  - 課題・レポート
  - 雰囲気
  - 指導医・スタッフ
  - 総合評価
- 自由記述レビュー
- 情報の最終更新日

## Content Style

- UI 文言は短く、説明しすぎない
- 専門職教育の文脈に合わせ、落ち着いた表現にする
- 「投稿」よりも `協力` `補完` `更新` の語彙を優先する

## Data Model for Demo

最低限、以下のモックデータを持つ。

- 実習先基本情報
- 構造化レビュー項目
- 自由記述レビュー
- AI 要約
- 不足項目
- ログイン時質問

## Implementation Notes

- 実装開始時は、画面遷移よりも `1 画面で価値が伝わること` を優先する
- まずは 3〜5 件の実習先ダミーデータで成立させる
- デザイン調整では、`カード密度` `可読性` `比較しやすさ` を重視する
- レスポンシブ対応は `desktop first` で設計しつつ、スマホでも崩さない
- `index.html` `manifest.webmanifest` `icon` の整合を保ち、PWA として破綻しないようにする
- 背景は静的なグラデーションとし、過度な常時アニメーションは避ける
- 強調アニメーションは局所的に使い、現在は `AI Summary` を病院切替時に 3 秒だけ波打たせる

## Working Rule For Future Changes

- 変更は小さく進める
- まず見た目と導線を作り、その後にモックロジックを足す
- 新しい画面を増やす前に、既存画面の情報設計を整える
- README などの説明文より、先に demo 画面の完成度を上げる
